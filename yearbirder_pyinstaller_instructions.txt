Here are the steps to bundle your yearbirder app with PyInstaller using a spec file:

**Prerequisites**
1. Make sure PyInstaller is installed: `pip install pyinstaller`
2. Activate your virtual environment if you're using one

**Bundling Steps**

1. Navigate to your project directory in the terminal:
   ```bash
   cd /path/to/yearbirder
   ```

2. Run PyInstaller with your spec file:
   ```bash
   pyinstaller yearbirder.spec
   ```

3. PyInstaller will generate two folders in your project directory:
   - `build/` — intermediate files (can be ignored)
   - `dist/` — contains your final app bundle (`yearbirder.app`)

4. Find your app bundle at:
   ```
   dist/yearbirder.app
   ```

**Optional Clean Rebuild**

If you want a clean build (recommended if you've made changes to the spec):
```bash
pyinstaller --clean yearbirder.spec
```

**Testing the Bundle**

5. Run the app directly from the terminal to see any error output:
   ```bash
   ./dist/yearbirder.app/Contents/MacOS/yearbirder
   ```

6. Or double-click `yearbirder.app` in Finder as a normal user would.

**If You Need to Re-sign (macOS Gatekeeper)**

7. If macOS blocks the app, you may need to ad-hoc sign it:
   ```bash
   codesign --force --deep --sign - dist/yearbirder.app
   ```

That's the core workflow. Since you have a spec file already, step 2 is really the heart of it — the spec captures all the configuration so you don't need to pass flags each time.
